# X 温和抓取补丁

这个补丁用于降低 X / Twitter 因连续访问、滚动次数过多或页面长时间加载导致的失败概率。

## 已调整

- GUI 抓取日志区域改为可扩展大面板，默认高度从 340 增加到 520。
- X 页面跳转超时后，如果浏览器已经进入目标页，不再立刻失败，而是继续尝试解析当前 DOM。
- 单次作者媒体解析默认滚动轮数从 12 降到 5。
- 单次媒体页最多收集 40 个媒体项。
- 两次 X 解析之间加入 25 秒冷却。
- 页面操作之间加入 3-8 秒随机延迟。
- 滚动改为小步随机滚动，并穿插轻量鼠标移动。
- 滚动时有 7% 概率停留 30-90 秒，模拟临时阅读/停顿。
- 下载 1 个文件后等待 1-3 秒，并叠加 0.8-1.5 抖动。
- 每下载 3-5 个文件后额外暂停 5-12 秒。
- GUI 的 X 模式会把“抓取数量”作为上限，默认 20，最高 120。
- 本地维护 `runtime/x_download_cache.json`，已下载 URL 会跳过，不重复请求。
- WebUI 新增本地 X 图片/视频缩略图墙，可删除选中的本地文件。
- 页面仍完全打不开时，错误提示改为中文可操作说明。

## 不做的部分

- 不实现伪装成大量不同真实用户的指纹轮换。
- 不通过高频代理或设备指纹规避平台检测。

当前策略以降低频率、减少规律性和复用登录会话为主，优先减少误伤和封号风险。

## 范围控制

- 作者媒体/搜索媒体只抓用户填写数量范围内的结果。
- WebUI 支持先预览再勾选下载。
- 本地已有文件和已记录 URL 会跳过。
- 不自动遍历全部页码，不后台静默拉完整 timeline。

## 建议

- 同一个作者页失败后等待 1-3 分钟再试。
- 不要连续快速点击「开始抓取」。
- X Cookie 失效或触发风控时，重新导入 `x.com_cookies.txt`。
- 大批量抓取建议先用 WebUI 的 X 作者媒体页预览和勾选下载。
